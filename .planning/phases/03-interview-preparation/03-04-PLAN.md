# Plan 03-04: Display Components — InterviewPrepPanel and QuestionCard

## Goal

Build the two display components for the interview preparation feature. `InterviewPrepPanel` orchestrates `useInterviewPrep` into phase-gated UI. `QuestionCard` is a stateless controlled component for one question: it receives all state as props and calls back parent-provided handlers. These mirror the relationship between `AnalysisPanel` and `RewriteDiff` from Phase 2.

## Context

**Depends on Plan 03-03** for the `useInterviewPrep` hook, `QuestionState`, and `PrepState` types. Wave 3 — the hook must exist before these components can be built.

**Existing shadcn components to reuse (do not rebuild):** `card`, `badge`, `button`, `progress`, `textarea`. All are already in `src/components/ui/`. Match the Tailwind style established across the app (`rounded-xl border bg-card`, `text-muted-foreground`, `font-semibold`, etc.).

**Component contracts:**

`InterviewPrepPanel` props:
```typescript
interface InterviewPrepPanelProps {
  resumeText: string
  jdText: string
}
```

`QuestionCard` props:
```typescript
interface QuestionCardProps {
  question: InterviewPrepResult['questions'][number]  // { question, category, rationale, modelAnswer }
  questionState: QuestionState                        // { expanded, draftAnswer, critiquePhase, critiqueText, critiqueError }
  index: number
  onToggle: () => void
  onDraftChange: (text: string) => void
  onSubmitCritique: () => void
}
```

`QuestionCard` holds NO internal state. All state lives in the hook via `InterviewPrepPanel`.

**Anti-pattern warnings (from RESEARCH.md):**
- Do NOT auto-start in a `useEffect` inside `InterviewPrepPanel`. The panel receives `resumeText` and `jdText` as props but does not call `startPrep` automatically. The CTA button in `page.tsx` (Plan 03-05) mounts this panel; the panel shows an idle UI with a "Generate interview questions" button. Clicking it calls `startPrep`.
- Do NOT use dynamic Tailwind class widths (`w-[${n}%]`). Any runtime-computed widths must use `style={{ width: ... }}` (STATE.md confirmed pitfall from Phase 2).

<task id="1" name="Create QuestionCard stateless controlled component">
Create `src/components/question-card.tsx` (`'use client'` — it has interactive controls).

Import from:
- `@/hooks/use-interview-prep` — `QuestionState` type
- `@/lib/schemas` — `InterviewPrepResult` for the question item type
- `@/components/ui/card`, `badge`, `button`, `textarea` — shadcn primitives

Props interface:
```typescript
interface QuestionCardProps {
  question: InterviewPrepResult['questions'][number]
  questionState: QuestionState
  index: number
  onToggle: () => void
  onDraftChange: (text: string) => void
  onSubmitCritique: () => void
}
```

Render structure:
- **Card header (always visible):** Question text (not truncated — show in full), `Badge` showing `question.category` (label it as-is: 'behavioural', 'technical', etc.), and a chevron icon (`ChevronDown` / `ChevronUp` from `lucide-react`) that calls `onToggle`. The entire header row is clickable (call `onToggle` on click).

- **Card body (only when `questionState.expanded === true`):**
  - **Rationale section:** A muted label "Why this question" and `question.rationale` text below it. This is the JD-grounding visibility — show it prominently.
  - **Model answer section:** A label "Model answer" (bold) and `question.modelAnswer` text. Style it distinctly (e.g., `bg-secondary` block or muted border-left treatment) so users can read it before writing their draft.
  - **Draft textarea:** Label "Your answer", then a `Textarea` component with `value={questionState.draftAnswer}` and `onChange={(e) => onDraftChange(e.target.value)}`. Placeholder: "Write your answer here…". Minimum height via `className="min-h-[120px]"`.
  - **Feedback button:** "Get feedback" when `questionState.critiquePhase === 'idle'` or `'error'`. "Revise feedback" when `questionState.critiquePhase === 'done'`. Disabled when `questionState.critiquePhase === 'streaming'`. Disabled when `questionState.draftAnswer.trim().length < 20` (the Zod minimum — do not call the API with too-short text; show a muted hint). On click → calls `onSubmitCritique`.
  - **Critique display (when `critiquePhase !== 'idle'`):**
    - `streaming`: a heading "Getting feedback…" and the accumulating `critiqueText` in a monospace pre-like block.
    - `done`: a heading "Feedback" and the full `critiqueText` rendered — use `whitespace-pre-wrap` so markdown line breaks appear. Do NOT parse the markdown; the system prompt produces clean bold headers and short paragraphs that render acceptably as preformatted text.
    - `error`: a muted red error message from `critiqueError`.

**Verify:** `npx tsc --noEmit` passes. `QuestionCard` has no internal state (`useState`, `useReducer`). All state comes from `questionState` prop.
</task>

<task id="2" name="Create InterviewPrepPanel orchestrating the hook" depends_on="1">
Create `src/components/interview-prep-panel.tsx` (`'use client'`). Props: `{ resumeText: string; jdText: string }`.

Call `useInterviewPrep()`. Render based on `state.phase`:

**idle:** A section heading like "Interview Preparation" and a "Generate interview questions" `Button` that calls `startPrep(resumeText, jdText)` on click. Do NOT auto-call in a `useEffect`.

**streaming:** A heading "Preparing your questions…", a `Progress` bar (indeterminate — `value={null}`), and a scrollable `div` (or `pre`) showing `state.progress` as it streams (monospace, muted, `max-h-48 overflow-y-auto`). This guarantees no blank screen during the 15-30s generation window.

**error:** An `Alert` with `variant="destructive"`, a heading "Failed to generate questions", the `state.message`, and a "Try again" `Button` that calls `startPrep(resumeText, jdText)`.

**done:** Render the full results in this order:
1. **Prep strategy block** — a card or styled section with:
   - `state.result.prepStrategy.seniorityLevel` (bold label + value)
   - `state.result.prepStrategy.domainContext` (label + value)
   - Tips rendered as a simple list (`state.result.prepStrategy.tips.map((tip, i) => ...)`) — no bullet points from code, the tips are prose strings.
2. **Question list heading** — "Interview Questions" with a count (`{state.result.questions.length} questions`).
3. **QuestionCard per question** — `state.result.questions.map((question, i) => ...)`. For each, pass:
   - `question={question}`
   - `questionState={state.questions[i]}`
   - `index={i}`
   - `onToggle={() => toggleExpand(i)}`
   - `onDraftChange={(text) => setDraft(i, text)}`
   - `onSubmitCritique={() => submitCritique(i, question.question, question.modelAnswer, state.questions[i].draftAnswer)}`
   
   Note that `submitCritique` here passes `state.questions[i].draftAnswer` from the panel's render scope — this is safe because the panel re-renders on every `setDraft` dispatch, so `state.questions[i].draftAnswer` is always current at the time the button is clicked. This is distinct from a stale callback closure.
   - `key={i}`

**Verify:** `npx tsc --noEmit` passes. The four phase branches (idle/streaming/error/done) are all present. `startPrep` is NOT called in a `useEffect`.
</task>
