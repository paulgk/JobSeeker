# Plan 03-05: Integration ã AnalysisPanel Callback and Page Wiring

## Goal

Wire the interview preparation feature into the page. Two surgical changes: (1) add an optional `onInterviewPrepReady` callback prop to `AnalysisPanel` ã when analysis is `done`, a "Prepare for interview" button calls it; (2) update `page.tsx` to hold a `showInterviewPrep` boolean flag ã when true, mount `InterviewPrepPanel` below `AnalysisPanel`. Wave 4 ã depends on the routes (03-02) and the components (03-04).

## Context

**Depends on:**
- Plan 03-02 ã API routes exist (runtime dependency via `useInterviewPrep` ã `fetch`)
- Plan 03-04 ã `InterviewPrepPanel` and `QuestionCard` components

**Existing files to modify MINIMALLY:**
- `src/components/analysis-panel.tsx` ã add ONE optional prop (`onInterviewPrepReady?: () => void`) and ONE button in the `done` phase footer. No other changes. The existing layout, all four result views, and the accept/reject flow are untouched.
- `src/app/page.tsx` ã add ONE `useState` flag (`showInterviewPrep`) and ONE conditional render below the existing `{analysing && <AnalysisPanel ... />}` block. Pass the new callback prop. No other changes to the page layout.

**Current page.tsx state (verified):** The file is already a `'use client'` component with `resumeText`, `jdText`, `analysing` state, and a conditional `<AnalysisPanel resumeText={resumeText} jdText={jdText} />`. Adding `showInterviewPrep` state and the `InterviewPrepPanel` conditional requires no architectural change.

**UX decision:** The "Prepare for interview" CTA lives inside `AnalysisPanel` (not in `page.tsx`). This avoids lifting analysis-phase awareness into `page.tsx`. When the user clicks it, `AnalysisPanel` calls `onInterviewPrepReady()`; `page.tsx` sets `showInterviewPrep = true`; `InterviewPrepPanel` mounts below. The analysis results stay visible.
