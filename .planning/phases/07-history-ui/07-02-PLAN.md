## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → PATCH /api/applications/[id]/status | Untrusted JSON body + session cookie + path param cross into route handler |
| route handler → DAL | Route handler must pass session.userId (not body.userId or any client-supplied id) to updateApplicationStatus |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06 | Spoofing | PATCH route auth | mitigate | verifySession() called as first step; throws on no session and handler returns 401 with no body. Pattern copied verbatim from src/app/api/analyse/route.ts which is verified in production. |
| T-07-07 | Elevation of Privilege | PATCH route IDOR | mitigate | userId is read from session (never from request body or URL). updateApplicationStatus(userId, id, status) uses `and(eq(applications.id, id), eq(applications.userId, userId))` WHERE clause — a request for a row owned by another user silently affects zero rows. Verified in dal.ts lines 74-83. |
| T-07-08 | Tampering | Invalid status enum injection | mitigate | VALID_STATUSES whitelist check before DAL call. Drizzle does not validate enum values at the ORM layer — Postgres would throw a 500 — explicit pre-check returns 400 with a clean error message. |
| T-07-09 | Tampering | Malformed JSON body | mitigate | try/catch around request.json() returns 400 `{ error: 'Invalid JSON' }`. Pattern copied from analyse/route.ts lines 57-64. |
| T-07-10 | Denial of Service | Unbounded request body | accept | proxy.ts rate-limits all /api/* routes to 20 req/min/IP (verified in proxy.ts lines 10-32). PATCH body is a tiny JSON object — no separate body size cap needed beyond Next.js defaults. |
| T-07-11 | Information Disclosure | Error body leaking row data | mitigate | Handler returns either empty 204 or `{ error: '<static string>' }` — never echoes the body or DB state. updateApplicationStatus returns void, so there is no DAL output to leak. |
| T-07-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this plan. Package Legitimacy Audit (07-RESEARCH.md) confirms Phase 7 adds no runtime npm packages directly — only shadcn-generated source files (handled in Plan 07-01). |
