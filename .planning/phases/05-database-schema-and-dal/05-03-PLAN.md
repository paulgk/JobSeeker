<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| route handler / RSC (caller) → DAL | Caller passes userId from a verified session; DAL must scope every query to that userId |
| client → application data | A user must never read or mutate another user's application row |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Elevation of Privilege (IDOR) | getApplicationById, updateApplicationStatus | mitigate | Every ownership-scoped query filters by `and(eq(applications.id, id), eq(applications.userId, userId))` — never by id alone (Security Domain: single most important concern) |
| T-05-06 | Tampering (SQL injection) | all DAL queries | mitigate | Drizzle query builder uses parameterized queries; no `sql` template with raw user input |
| T-05-07 | Information Disclosure | getApplications list query | mitigate | D-09 metadata-only select; blob columns (resumeText/jdText/analysisData/interviewData) excluded from list query; dal.ts is `server-only` |
| T-05-08 | Information Disclosure | DATABASE_URL exposure | mitigate | db imported only into server-only dal.ts/auth.ts; DATABASE_URL never prefixed NEXT_PUBLIC_ |
</threat_model>
