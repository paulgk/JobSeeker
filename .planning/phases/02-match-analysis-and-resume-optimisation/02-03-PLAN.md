# Plan 02-03: Client Data Layer — Diff Utility and SSE Hook

## Goal

Build the two client-side data primitives the UI sits on: an `InlineDiff` component (`src/lib/diff.ts`) that renders word-level before/after highlighting using the `diff` package, and a `useAnalysis` hook (`src/hooks/use-analysis.ts`) that POSTs to `/api/analyse`, consumes the SSE stream progressively via `fetch` + `ReadableStreamDefaultReader`, and manages all client state (streaming progress, final result, and per-rewrite accept/reject) with `useReducer`. No persistence — v1 is stateless.

## Context

**Depends on Plan 02-01** which installed the `diff` package and added `AnalysisResult` (and the `result` SSE event) to schemas.ts. This plan does NOT depend on the route (Plan 02-02) at build time — the hook talks to the route via `fetch` at runtime, so they can be built in parallel in Wave 2.

**Why `fetch` not EventSource:** The route is POST; `EventSource` is GET-only. Use `fetch` + `res.body.getReader()` + `TextDecoder`, splitting on `\n\n` and buffering incomplete lines between reads. (RESEARCH.md "SSE Client-Side Consumption".)

**State shape (RESEARCH.md Pattern 5):**
```typescript
type RewriteState = { status: 'pending' | 'accepted' | 'rejected'; section: RewriteSection }

type AnalysisState =
  | { phase: 'idle' }
  | { phase: 'streaming'; progress: string }
  | { phase: 'done'; result: AnalysisResult; rewrites: RewriteState[] }
  | { phase: 'error'; message: string }
```
Reducer actions: `STREAM_START`, `CHUNK` (append to progress), `RESULT` (transition to done, seed rewrites as 'pending'), `ERROR`, `ACCEPT_REWRITE(index)`, `REJECT_REWRITE(index)`.

**Pitfall:** Do NOT `JSON.parse` partial chunks. `chunk` events are raw progress text appended to `progress`. The structured object arrives only on the `result` event.

```typescript
'use client'

import { diffWords, type Change } from 'diff'

export function InlineDiff({ original, rewritten }: { original: string; rewritten: string }) {
  const parts: Change[] = diffWords(original, rewritten)
  return (
    <span className="font-mono text-sm leading-relaxed whitespace-pre-wrap">
      {parts.map((part, i) => (
        <span
          key={i}
          className={
            part.added
              ? 'bg-green-100 text-green-800 rounded px-0.5'
              : part.removed
              ? 'bg-red-100 text-red-800 line-through rounded px-0.5'
              : 'text-foreground'
          }
        >
          {part.value}
        </span>
      ))}
    </span>
  )
}
```

<task id="2" name="Create useAnalysis hook with reducer and SSE consumer" depends_on="1">
Create `src/hooks/use-analysis.ts` (`'use client'`). Import the `AnalysisResult` type and (for the rewrite section type) reuse the schema-inferred shape from `@/lib/schemas`. Implement the reducer with the state shape and actions above.

`useAnalysis()` returns `{ state, start, acceptRewrite, rejectRewrite }`:
- `start(resumeText: string, jdText: string)` dispatches `STREAM_START`, then POSTs to `/api/analyse` and consumes the stream:

```typescript
const res = await fetch('/api/analyse', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ resumeText, jdText }),
})
if (!res.ok || !res.body) { dispatch({ type: 'ERROR', message: 'Failed to connect' }); return }
const reader = res.body.getReader()
const decoder = new TextDecoder()
let buffer = ''
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  buffer += decoder.decode(value, { stream: true })
  const lines = buffer.split('\n\n')
  buffer = lines.pop() ?? ''
  for (const line of lines) {
    if (!line.startsWith('data: ')) continue
    const event = JSON.parse(line.slice(6))
    if (event.type === 'chunk') dispatch({ type: 'CHUNK', content: event.content })
    if (event.type === 'result') dispatch({ type: 'RESULT', data: event.data })
    if (event.type === 'error') dispatch({ type: 'ERROR', message: event.message })
  }
}
```

- On `RESULT`, seed `rewrites` from `result.rewrites` each as `{ status: 'pending', section }`.
- `acceptRewrite(index)` / `rejectRewrite(index)` only mutate the matching entry's `status`; they are no-ops unless `state.phase === 'done'`.

Export the `RewriteState` type and a `RewriteSection` type alias so the rewrite component (Plan 02-05) and panel (Plan 02-06) can consume them without re-deriving. Define `RewriteSection` as the element type of `AnalysisResult['rewrites']`.

- The hook references `/api/analyse` (grep confirms) and uses `getReader()` + buffer split on `\n\n`.
- `acceptRewrite`/`rejectRewrite` are exposed and guarded by `phase === 'done'`.
