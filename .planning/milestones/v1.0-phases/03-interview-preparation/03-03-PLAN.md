# Plan 03-03: Client Data Layer — use-interview-prep Hook

## Goal

Build `use-interview-prep.ts`: the single source of truth for all interview preparation state. The hook mirrors the architecture of `use-analysis.ts` but extends it with per-question state arrays for independent expand/collapse, draft text, and inline streaming critique. This plan does NOT depend on the API routes (Plan 03-02) at build time — the hook talks to the routes via `fetch` at runtime — so Plans 03-02 and 03-03 run in parallel in Wave 2.

## Context

**Depends on Plan 03-01** for `InterviewPrepResult` type in `src/lib/schemas.ts`. Does NOT require the routes (03-02) — `fetch` calls happen at runtime, not import time.

**Why a separate hook (not extending use-analysis.ts):** The analysis hook uses a single per-phase state object. Interview prep needs per-question arrays — adding them to `use-analysis.ts` would merge incompatible reducer shapes and create coupling between unrelated features. One hook, one concern.

**Primary pattern source:** `src/hooks/use-analysis.ts` — copy the SSE consumer loop and `useCallback` patterns exactly.

**Critical pitfalls (from RESEARCH.md):**

1. **Per-question state mutation:** The reducer must return a NEW array via `.map()` for any per-question update. Never mutate `state.questions[index]` directly. Follow the `ACCEPT_REWRITE` pattern from `use-analysis.ts` exactly.

2. **CRITIQUE_START must reset critiqueText:** When a user re-submits critique on an already-critiqued question, `CRITIQUE_START` must reset both `critiquePhase` to `'streaming'` AND `critiqueText` to `''` (and `critiqueError` to `''`). Without this, new chunks append to old critique text.

3. **submitCritique closure staleness:** `submitCritique` must read `draftAnswer` from its own function parameter (passed by the component), NOT from `state.questions[index].draftAnswer` captured in a stale closure. The component reads the textarea's current value and passes it in. The callback signature is `submitCritique(index, question, modelAnswer, draftAnswer)` — four arguments, not three.

4. **QUESTIONS_RESULT initialises the questions array:** On receiving the `result` SSE event from `/api/interview-questions`, dispatch `QUESTIONS_RESULT` with `data: InterviewPrepResult`. The reducer creates `questions` as an array of `QuestionState` objects, one per `data.questions` entry, all initialised to `{ expanded: false, draftAnswer: '', critiquePhase: 'idle', critiqueText: '', critiqueError: '' }`.

5. **Do NOT auto-start on mount:** `useAnalysis` auto-starts via `useEffect` in `AnalysisPanel`. `InterviewPrepPanel` must NOT auto-start — the user explicitly clicks the CTA. `startPrep` is only called from the button's `onClick`.

```typescript
  | { phase: 'done'; result: InterviewPrepResult; questions: QuestionState[] }
  | { phase: 'error'; message: string }
```

**Reducer actions (all ten — do not omit any):**

```typescript
type Action =
  | { type: 'STREAM_START' }
  | { type: 'CHUNK'; content: string }
  | { type: 'QUESTIONS_RESULT'; data: InterviewPrepResult }
  | { type: 'ERROR'; message: string }
  | { type: 'TOGGLE_EXPAND'; index: number }
  | { type: 'SET_DRAFT'; index: number; text: string }
  | { type: 'CRITIQUE_START'; index: number }
  | { type: 'CRITIQUE_CHUNK'; index: number; content: string }
  | { type: 'CRITIQUE_DONE'; index: number }
  | { type: 'CRITIQUE_ERROR'; index: number; message: string }
```

**Reducer implementation — each case must exactly follow these rules:**
- `STREAM_START` → `{ phase: 'streaming', progress: '' }`
- `CHUNK` → guard `state.phase !== 'streaming'` then append: `{ ...state, progress: state.progress + action.content }`
- `QUESTIONS_RESULT` → transition to `done`: `{ phase: 'done', result: action.data, questions: action.data.questions.map(() => ({ expanded: false, draftAnswer: '', critiquePhase: 'idle', critiqueText: '', critiqueError: '' })) }`
- `ERROR` → `{ phase: 'error', message: action.message }`
- `TOGGLE_EXPAND` → guard `state.phase !== 'done'`; map over `questions`, flip `expanded` at `action.index` only
- `SET_DRAFT` → guard `state.phase !== 'done'`; map over `questions`, set `draftAnswer` at `action.index` only
- `CRITIQUE_START` → guard `state.phase !== 'done'`; map over `questions`, reset at `action.index` to `{ ...q, critiquePhase: 'streaming', critiqueText: '', critiqueError: '' }` — this reset is mandatory for re-submission
- `CRITIQUE_CHUNK` → guard `state.phase !== 'done'`; map over `questions`, append `action.content` to `critiqueText` at `action.index` only
- `CRITIQUE_DONE` → guard `state.phase !== 'done'`; map over `questions`, set `critiquePhase: 'done'` at `action.index`
- `CRITIQUE_ERROR` → guard `state.phase !== 'done'`; map over `questions`, set `{ critiquePhase: 'error', critiqueError: action.message }` at `action.index`

**Hook implementation:**

```typescript
export function useInterviewPrep() {
  const [state, dispatch] = useReducer(reducer, { phase: 'idle' })

  const startPrep = useCallback(async (resumeText: string, jdText: string) => {
    dispatch({ type: 'STREAM_START' })
    let res: Response
    try {
      res = await fetch('/api/interview-questions', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ resumeText, jdText }),
      })
    } catch {
      dispatch({ type: 'ERROR', message: 'Network error — could not connect' })
      return
    }
    if (!res.ok || !res.body) {
      dispatch({ type: 'ERROR', message: 'Failed to connect' })
      return
    }
    // SSE consumer loop — copy from use-analysis.ts exactly:
    const reader = res.body.getReader()
    const decoder = new TextDecoder()
    let buffer = ''
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      buffer += decoder.decode(value, { stream: true })
      const lines = buffer.split('\n\n')
      buffer = lines.pop() ?? ''
      for (const line of lines) {
        if (!line.startsWith('data: ')) continue
        let event: { type: string; content?: string; data?: InterviewPrepResult; message?: string }
        try { event = JSON.parse(line.slice(6)) } catch { continue }
        if (event.type === 'chunk' && event.content !== undefined)
          dispatch({ type: 'CHUNK', content: event.content })
        if (event.type === 'result' && event.data !== undefined)
          dispatch({ type: 'QUESTIONS_RESULT', data: event.data })
        if (event.type === 'error' && event.message !== undefined)
          dispatch({ type: 'ERROR', message: event.message })
      }
    }
  }, [])

  // draftAnswer passed in from component to avoid closure staleness (Pitfall 3)
  const submitCritique = useCallback(async (
    index: number,
    question: string,
    modelAnswer: string,
    draftAnswer: string
  ) => {
    dispatch({ type: 'CRITIQUE_START', index })
    let res: Response
    try {
      res = await fetch('/api/interview-critique', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ question, modelAnswer, draftAnswer }),
      })
    } catch {
      dispatch({ type: 'CRITIQUE_ERROR', index, message: 'Network error — could not connect' })
      return
    }
    if (!res.ok || !res.body) {
      dispatch({ type: 'CRITIQUE_ERROR', index, message: 'Failed to connect' })
      return
    }
    const reader = res.body.getReader()
    const decoder = new TextDecoder()
    let buffer = ''
    while (true) {
      const { done, value } = await reader.read()
      if (done) break
      buffer += decoder.decode(value, { stream: true })
      const lines = buffer.split('\n\n')
      buffer = lines.pop() ?? ''
      for (const line of lines) {
        if (!line.startsWith('data: ')) continue
        let event: { type: string; content?: string; message?: string }
        try { event = JSON.parse(line.slice(6)) } catch { continue }
        if (event.type === 'chunk' && event.content !== undefined)
          dispatch({ type: 'CRITIQUE_CHUNK', index, content: event.content })
        if (event.type === 'done')
          dispatch({ type: 'CRITIQUE_DONE', index })
        if (event.type === 'error' && event.message !== undefined)
          dispatch({ type: 'CRITIQUE_ERROR', index, message: event.message })
      }
    }
  }, [])

  const toggleExpand = useCallback((index: number) => {
    dispatch({ type: 'TOGGLE_EXPAND', index })
  }, [])

  const setDraft = useCallback((index: number, text: string) => {
    dispatch({ type: 'SET_DRAFT', index, text })
  }, [])

  return { state, startPrep, submitCritique, toggleExpand, setDraft }
}
```
