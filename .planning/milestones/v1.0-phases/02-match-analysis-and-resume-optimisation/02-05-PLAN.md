# Plan 02-05: Rewrite Diff with Accept/Reject

## Goal

Build the `RewriteDiff` component: for a single resume-section rewrite, show the section name, a before/after word-level diff (via `InlineDiff`), and per-section Accept / Reject controls. The component is controlled â€” it reflects the `status` from `RewriteState` and calls back `onAccept` / `onReject` so the parent panel (Plan 02-06) owns the state via the `useAnalysis` reducer. This satisfies success criterion 4 (review each rewrite, see exactly what changed, accept or discard individually).

## Context

**Depends on Plan 02-03** for `InlineDiff` (`src/lib/diff.tsx`) and the `RewriteState` / `RewriteSection` types (`src/hooks/use-analysis.ts`). Wave 3 (the diff util must exist first).

**Controlled component contract** â€” state lives in the parent (the `useAnalysis` reducer), not here:
```typescript
interface RewriteDiffProps {
  rewrite: RewriteState          // { status: 'pending'|'accepted'|'rejected', section }
  onAccept: () => void
  onReject: () => void
}
```
The parent maps over `state.rewrites` and renders one `RewriteDiff` per entry, wiring `onAccept={() => acceptRewrite(i)}` and `onReject={() => rejectRewrite(i)}`.

Mark `'use client'` (it has interactive buttons). Reuse existing `button` (`variant="default"` for accept, `variant="outline"` for reject) and optionally `card`/`badge`/`separator`. Use `lucide-react` icons (`Check`, `X`) if helpful â€” already installed.

**Anti-hallucination is enforced upstream** (system prompt + verbatim `originalText`), so the diff is meaningful here. Do not re-validate content; just render the diff.
