# Plan 02-04: Display Components — Score, Action Items, Keyword Gaps

## Goal

Build the three pure presentational components that render the validated analysis: `ScoreCard` (overall score plus the four-component breakdown with rubric weights and rationale), `ActionList` (prioritised action items with impact badges), and `KeywordBadges` (JD keyword gaps in priority order). These satisfy success criteria 1, 2, and 3. They are stateless render-only components consuming typed props derived from `AnalysisResult`.

## Context

**Depends on Plan 02-01** for the `AnalysisResult` type (and its `components`, `actionItems`, `keywordGaps` member types) in `src/lib/schemas.ts`. No dependency on the route, hook, or diff util — these are leaf rendering components and can be built in parallel in Wave 2.

**Existing shadcn primitives to reuse (do not rebuild):** `card`, `badge`, `separator` (added in Plan 02-01). Match the existing Tailwind style seen in `page.tsx` (`rounded-xl border bg-card`, `text-muted-foreground`, `font-semibold`).

**Component contracts (keep props typed against schema slices, not `any`):**
- `ScoreCard` takes `{ overallScore: number; components: AnalysisResult['components'] }`.
- `ActionList` takes `{ items: AnalysisResult['actionItems'] }`.
- `KeywordBadges` takes `{ keywords: string[] }`.

These are presentational and need no `'use client'` unless they use hooks — they don't, so leave them as server-compatible components (the parent panel in Plan 02-06 is the client boundary). However, since they will be imported by a `'use client'` panel, either marking or not marking is fine; do NOT add interactivity here.

## Tasks

<task id="1" name="Create ScoreCard with overall score and component breakdown">
Create `src/components/score-card.tsx`. Render:
- A prominent overall score (e.g. `{overallScore}%`) at the top — large, bold.
- For each component (Required Skills, Experience Level, Domain Match, Education): the component `name`, its `weight` shown as a percentage (e.g. `weight * 100` → "40%"), its `score` (0-100), a simple horizontal bar whose width reflects `score` (Tailwind width via inline style `style={{ width: \`${score}%\` }}` on a filled div inside a track div), and the `rationale` text below.
- Wrap in the existing `Card` primitive. Use `Separator` between components if helpful.

Props: `{ overallScore: number; components: AnalysisResult['components'] }`. Import `AnalysisResult` from `@/lib/schemas`.

This is the "not a black-box number" requirement — every component must show its rationale.

**Verify:** `npx tsc --noEmit` passes; component renders overallScore and maps over components showing name, weight%, score, bar, rationale.
</task>

<task id="2" name="Create ActionList with ranked items and impact badges" depends_on="1">
Create `src/components/action-list.tsx`. Props: `{ items: AnalysisResult['actionItems'] }`.

Sort/display by `rank` ascending (rank 1 first). For each item show: the rank, `title` (bold), `detail` (full text — must be actionable without further guidance, so do not truncate), and an impact `Badge` whose variant/color reflects `estimatedImpact` (high → a strong/destructive-or-default color, medium → secondary, low → outline). Use the existing `badge` component; map impact → variant in a small lookup.

Wrap in a `Card` with a heading like "Action Items". Handle the empty array case (render a muted "No action items" line) so it never crashes on an empty list.

**Verify:** `npx tsc --noEmit` passes; items render rank, title, full detail, and an impact badge; empty array renders gracefully.
</task>

<task id="3" name="Create KeywordBadges for keyword gaps" depends_on="1">
Create `src/components/keyword-badges.tsx`. Props: `{ keywords: string[] }`.

Render the keywords as a wrapped flex of `Badge` components in the order received (the API already orders by priority — do NOT re-sort). Wrap in a `Card` with a heading like "Keyword Gaps" and a short subtitle ("From the job description, missing in your resume — add where truthful."). Handle the empty array case with a muted "No significant keyword gaps found." message.

**Verify:** `npx tsc --noEmit` passes; keywords render as badges in original order; empty array renders the muted fallback.
</task>
