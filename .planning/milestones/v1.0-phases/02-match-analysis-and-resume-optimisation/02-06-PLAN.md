# Plan 02-06: Integration — Analysis Panel and Page Wiring

## Goal

Wire everything together. Create `AnalysisPanel` (the `'use client'` results container that drives `useAnalysis` and renders progress, errors, and the four result views) and update `page.tsx` to capture `resumeText` and `jdText` from the existing `ResumePanel` / `JobDescriptionPanel` `onReady` callbacks into shared state, then render `AnalysisPanel` with an "Analyse" trigger. This closes the phase: a user with both inputs gets streamed, structured results they can act on and selectively accept. Wave 4 — depends on the route, hook, and all components.

## Context

**Depends on:**
- Plan 02-02 — the real `/api/analyse` route (runtime dependency via the hook).
- Plan 02-03 — `useAnalysis` hook (`{ state, start, acceptRewrite, rejectRewrite }`) and `RewriteState`.
- Plan 02-04 — `ScoreCard`, `ActionList`, `KeywordBadges`.
- Plan 02-05 — `RewriteDiff`.

**Existing files (locked Phase 1 contracts):**
- `src/app/page.tsx` — currently a SERVER component rendering the two panels side by side. It must become a client component (or delegate to a small client wrapper) to hold `resumeText`/`jdText` state from the `onReady` callbacks. STATE.md decision: "page.tsx stays server component; ResumePanel carries 'use client' as the client boundary." To honor the spirit (keep the page shell simple) while adding shared state, introduce a thin `'use client'` wrapper component for the interactive area rather than converting the whole page if you prefer — but converting `page.tsx` to a client component is acceptable here since it now needs interactivity. Pick the simpler path and note it in the SUMMARY.
- `ResumePanel` and `JobDescriptionPanel` already accept `onReady?: (text: string) => void`. Use these — do NOT modify the panels.

**UX flow:** resume `onReady` → set `resumeText`; JD `onReady` → set `jdText`. When BOTH are non-empty, enable an "Analyse" button. Clicking calls `start(resumeText, jdText)`. While `phase === 'streaming'`, show a `Progress` bar (indeterminate-style is fine) plus the streaming `progress` text (a "thinking" feed) so there is never a blank wait (success criterion 5). On `phase === 'done'`, render the four views. On `phase === 'error'`, render an `Alert` with the message.

## Tasks

<task id="1" name="Create AnalysisPanel orchestrating useAnalysis into the result views">
Create `src/components/analysis-panel.tsx` (`'use client'`). Props: `{ resumeText: string; jdText: string }`.

Use `useAnalysis()`. Render based on `state.phase`:
- `idle`: an "Analyse Match" button, disabled unless both `resumeText` and `jdText` are non-empty. On click → `start(resumeText, jdText)`.
- `streaming`: a `Progress` bar + a scrollable area (use shadcn `scroll-area`) showing `state.progress` as it streams (monospace, muted). A heading like "Analysing…". This guarantees no blank screen.
- `done`: render in order — `ScoreCard` (overallScore + components), `ActionList` (actionItems), `KeywordBadges` (keywordGaps), then a "Suggested Rewrites" section mapping `state.rewrites` to `RewriteDiff` with `onAccept={() => acceptRewrite(i)}` / `onReject={() => rejectRewrite(i)}`. Use `Separator` between sections. Allow re-running (a "Re-analyse" button is optional).
- `error`: an `Alert variant="destructive"` with `state.message` and a retry button that calls `start` again.

Import the components from Plans 02-04 / 02-05 and the hook from 02-03.

**Verify:** `npx tsc --noEmit` passes; all four result views and `RewriteDiff` are rendered in the `done` branch; streaming branch shows progress; error branch shows the alert.
</task>

<task id="2" name="Wire page.tsx to capture inputs and render AnalysisPanel" depends_on="1">
Update `src/app/page.tsx` so the interactive area holds `resumeText` and `jdText` state, fed by the panels' `onReady` callbacks, and renders `AnalysisPanel` below the two-panel grid.

Simplest path: convert `page.tsx` to `'use client'` and add `const [resumeText, setResumeText] = useState('')` and `const [jdText, setJdText] = useState('')`. Do NOT modify `ResumePanel` or `JobDescriptionPanel` internals — only pass the `onReady` props.

**Verify:** `npx tsc --noEmit` and `npm run lint` pass; `page.tsx` passes `onReady` to both panels and renders `AnalysisPanel` with both texts.
