## Milestone Goal

Replace the stateless one-shot session with a Google-authenticated app that saves every analysis so users can revisit match results, accepted rewrites, and interview prep weeks after the original run.

## Key Accomplishments

1. **Google OAuth + email/password auth** — better-auth 1.6.11, proxy.ts session guard for /history, inline verifySession() on all three LLM SSE routes (HOC wrapper skipped: breaks ReadableStream)
2. **Neon Postgres live** — Drizzle schema migrated (applications + better-auth tables), typed CRUD DAL with IDOR double-filter (`and(eq(id, id), eq(userId, userId))`), drizzleAdapter replacing memoryAdapter
3. **Auto-save after analysis** — Haiku-based job title/company extraction, analysis persisted to DB on SSE completion, applicationId propagated through full SSE client chain
4. **History UI** — /history list (metadata-only query, no JSONB blobs loaded), detail page with tabbed read-only analysis + interview prep display, StatusSelect with optimistic update, PATCH routes for status changes
5. **Re-run flow** — GET /api/applications/[id]/prefill, initialValue props on both input panels, home page searchParams pre-population via use(searchParams)
6. **History enhancements** (Phase 8 bonus) — click-to-edit company/role inline (blur/Enter auto-save), amber "Edit needed" badge for unknown values, inline InterviewPrepIsland on detail page, 401→/sign-in redirect fix, save_error Alert visibility

## Key Decisions Made This Milestone

| Decision | Outcome |
|----------|---------|
| better-auth over next-auth | next-auth has blocking peer dep conflict with Next.js 16; better-auth 1.6.x is current replacement |
| @neondatabase/serverless over @vercel/postgres | @vercel/postgres discontinued Dec 2024; Neon HTTP driver is stateless-correct for serverless |
| JWT sessions (not DB sessions) | Single-user tool; no server-side revocation needed; avoids sessions table |
| JSONB columns for analysis and interview data | Maps directly to existing Zod types; zero migration risk as prompts evolve |
| verifySession() inline (not HOC wrapper) | auth() HOC wrapper breaks ReadableStream — documented in Auth.js issue #12485 |
| proxy.ts replaces middleware.ts | Next.js 16 hard requirement |
| Metadata-only query for history list | Never select resumeText/jdText/JSONB blobs in list query — performance constraint |
| Auto-save on SSE completion | Save triggered automatically when analysis phase === 'done'; no explicit SaveButton |
| Read-only components for history detail | AnalysisPanel is wired to live SSE; separate display-only components built |
| Re-run via navigation (?applicationId=) | No new API surface; pre-populate existing page.tsx form state from DB values |
| window.location.href for 401 redirect | SSE hooks are 'use client' but have no router instance; hard navigation is correct |
